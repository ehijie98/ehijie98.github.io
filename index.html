Semantic HTML

Semantic essential means bringing meaning into the code that we write. With Semantic HTML, you are introducing elements that tell developers and browsers exactly what it does and what content belongs in it.
Semantic HTML introduces elements that can tell developers exactly what the element does or where itâ€™s placed based on the name of that element. Some of these elements are <header>, <nav>, <main>, and <footer>

Semantic HTML tag Semantic HTML name Meaning
<video> Video Allows us to add videos to our website.
<audio> Audio Allows us to add audio to our website.
<embed> Embed Allows us to add any type of media to our website.
<figure> and <figcaption> Figure and Figure Caption A way of encapsulating media, using both elements to group the media and description.
<section> and <article> Section and Article Though both do not have to be used together. <section> defines elements in a document, such as chapters, headings, or any other area of the document with the same theme.
<article> holds content that makes sense on its own such as articles, blogs, and comments. Generally, <section> can be used to define a theme for the webpage and <article> can be used to write independent content for that theme.
<aside> Aside This element is used to mark additional information that can enhance another element but isn't required in order to understand the main content.
